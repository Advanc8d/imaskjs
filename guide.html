Getting Started

Include imask.js or imask.min.js from dist folder:

<script src="dist/imask.js"></script>

Simple usage case:

var element = document.getElementById('selector');
var maskOptions = {
  mask: '+{7}(000)000-00-00'
};
var mask = new IMask(element, maskOptions);

Since version 1.0.0 mask was splitted to model and view layers.
Model layer contains all masking facilities which could be used independently with no UI.
View layer is just a glue between UI component and model, it connects listeners and controls changes in both directions.

View layer currently consist only of a single component to support input-like API. Instance of that type is returned by calling IMask. Basic use case:

var mask = new IMask(element, maskOptions);

// get/set value and unmasked value
mask.value = "+7(999)999-99-99";
console.log(mask.value);  // logs "+7(999)999-99-99"
console.log(mask.unmaskedValue);  // logs "79999999999"

mask.unmaskedValue = "70000000000";
console.log(mask.value);  // logs "+7(000)000-00-00"
console.log(mask.unmaskedValue);  // logs "70000000000"

// get/set mask
mask.mask = "999-999"; // auto-updates UI

// update options
mask.updateOptions({
  mask: Number,
  min: 0,
  max: 100
});  // also updates UI

// clean and destroy
mask.destroy();

// TODO events accept, complete

// get masked model
var masked = mask.masked;
masked.reset(); // UI will NOT be updated

Finally mask automatically start listening for element events. For manual control use bindEvents/unbindEvents methods.

Events

Subscription for accept, complete events are only available with pattern mask type or can be fired manually with fireEvent (event) method.

Use on (event, handler) method for subscribe.

Use off (event [, handler]) for unsibscribe. Calling without handler argument removes all attached handlers.

In example above any changes are proxied to model layer at first and then UI gets updated.

The core of masking on model layer is IMask.Masked base class. There are also several other model classes which corresponds to mask property type and provides additional functionality.

Common

IMask.Masked is a base class of all other *Masked. When you call IMask with mask options they are just directly passed to Masked model.

Usually you don't need manually create instances of that type, because it will be done by IMask internally. But you can subclass from it and add some specific behavior like uppercasing for example.

Additionaly to mask option you could pass custom validator as validate option for some complex checks on any mask types except Function and RegExp, because they are already validators itself.

Make sure that your mask or validator not just accept final value, but also any of intermediate states. Otherwise value will be cleared.

Example usage:

var masked = new IMask.Masked({
  mask: /^\d+$/,  // mask is mandatory
  validate: function (value, masked) {  // optional validator
    // let restrict input to start with '2'
    return value.indexOf('2') === 0;
  }
});

get/set mask (only same type allowed!)

masked.mask = /^\s+$/;  // ok
masked.mask = "0000";  // ERROR! changing mask type on existing mask is not allowed!

get/set value and unmasked value

masked.value = 'hello world!';
console.log(masked.unmaskedValue);

override commit method to make some postprocessing after UI is deactivated

String (Pattern)
use pattern for complex masks: 
mask: '000[aaa]/{NIC-}*[**]'
where:
  • 0 - any digit input
  • a - any letter input
  • * - any char input
  • other chars (not defined in custom options) supposed to be fixed
  • [] - make input optional
  • {} - include fixed in unmasked value
  • ' - prevent symbols shift back

If definition character should be treated as fixed it should be escaped by \\ (E.g. \\0).

Configure with options:

placeholder: {
lazy: boolean
char: <any single char>  // defaults to '_'
}
definitions: {
<any single char>: <same type as mask (RegExp, Function*, etc.)>
...
// defaults are '0', 'a', '*'
}

*Function signature differs from base mask: function (char, charIndex, str).

Mask type used to build a resolver (or inherit BaseMask). Resolver should define at least one method resolve (input, details) {...}.

Details is an object:

{
  cursorPos: <current cursor position>,
  oldSelection: {
    start: <index of the first previously selected character>,
    end: <index of the character after the last previously selected character>
  },
  oldValue: <previous user input>
}

Examples

Phone number

var phoneMask = new IMask(document.getElementById('phone-mask'), {
  mask: '+{7}(000)000-00-00'
}).on('accept', function() {/* ... */})
.on('complete', function() {/* ... */});

Only numbers

var numbersMask = new IMask(document.getElementById('numbers-mask'), {
  mask: function (str, details) {
    var inserted = details.inserted;
    if (!inserted.length) return true;
    var insertedNumbers = inserted.replace(/\D/g, '');

    details.cursorPos -= inserted.length - insertedNumbers.length;
    return details.head + insertedNumbers + details.tail;
  }
});

Pipe uppercase

var uppercaseMask = new IMask(document.getElementById('uppercase-mask'), {
  mask: [
    {
      mask: "aaa"
    },
    {
      mask: function (str) {
        return str.toUpperCase();
      }
    }
  ]
});
